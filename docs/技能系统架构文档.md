## 一、整体架构

### 1.1 目录结构

```
src/renderer/core/
├── skills/                          # ★ 技能库
│   ├── index.ts                     # 主入口（注册/调用）
│   ├── types.ts                     # 类型定义
│   ├── registry.ts                  # 技能管理器
│   │
│   ├── search-browser/              # 技能：打开浏览器搜索
│   │   └── index.ts                 # + SKILL.md 文档
│   │
│   ├── search-web/                  # 技能：网页搜索（SearXNG）
│   │   └── index.ts                 # + SKILL.md 文档
│   │
│   └── weather/                     # 技能：天气查询（wttr.in）
│       └── index.ts                 # + SKILL.md 文档
│
└── tools/                           # ★ 工具库
    └── index.ts                     # 基础工具定义
```

### 1.2 核心概念

| 概念 | 说明 | 例子 |
|------|------|------|
| **Tool（工具）** | 基础能力，只做一件事 | `browser.open-search` - 打开浏览器搜索 |
| **Skill（技能）** | 工具的组合，完成复杂任务 | `search-browser` - 打开浏览器搜索技能 |
| **Intent（意图）** | 用户想要做什么 | `search_web` - 搜索网页 |

### 1.3 调用流程

```
用户输入："搜索 B 站"
    ↓
意图识别：search_web
    ↓
TaskExecutor：执行层
    ↓
executeSkillViaNewSystem：调用技能系统
    ↓
skillExecutor.execute('search-web', { query: 'B 站' })
    ↓
searchWebSkill.execute()
    ↓
callTool('network.searxng-search', { query: 'B 站' })
    ↓
SearXNG API 调用
    ↓
返回结果 → 格式化 → 展示给用户
```

## 二、核心组件

### 2.1 类型定义（types.ts）

```typescript
// 工具定义
interface ToolDefinition {
  id: string;              // 工具唯一标识
  name: string;            // 工具名称
  description: string;     // 描述
  category: string;        // 分类
  params: ToolParamDef[];  // 参数定义
  execute: (params) => Promise<ToolResult>;
}

// 技能定义
interface SkillDefinition {
  metadata: SkillMetadata;   // 元数据（YAML frontmatter）
  params: SkillParamDef[];   // 参数定义
  whenToUse: string[];       // 使用场景
  examples: SkillExample[];  // 示例
  errorHandling: ErrorHandlingRule[];  // 错误处理
  execute: (params, steps?) => Promise<SkillResult>;
}

// 技能执行步骤（用于可视化）
interface SkillExecutionStep {
  stepNumber: number;
  name: string;
  description: string;
  toolId?: string;
  result?: ToolResult;
  startTime: number;
  endTime: number;
  success: boolean;
}
```

### 2.2 技能管理器（registry.ts）

**skillRegistry** - 技能注册表（单例）
- `register(skill)` - 注册技能
- `getAll()` - 获取所有技能
- `getById(id)` - 根据 ID 获取
- `search(query)` - 搜索技能（根据意图匹配）
- `getStats()` - 获取统计信息

**skillExecutor** - 技能执行器（单例）
- `execute(skillId, params)` - 执行技能
- 自动记录执行步骤
- 支持可视化展示

**callTool()** - 辅助函数
- 在技能内部调用工具
- 自动记录步骤到 `steps` 数组

### 2.3 工具库（tools/index.ts）

已实现的工具：

| 工具 ID | 名称 | 功能 |
|---------|------|------|
| `browser.open-external` | 打开外部链接 | 在浏览器中打开 URL |
| `browser.open-search` | 打开搜索 | 打开搜索引擎并搜索 |
| `network.http-get` | HTTP GET | 发送 HTTP 请求 |
| `network.searxng-search` | SearXNG 搜索 | 使用本地 SearXNG 搜索 |
| `network.wttr-weather` | wttr.in 天气 | 查询天气（无需 API Key） |

## 三、技能开发指南

### 3.1 创建新技能的步骤

**第 1 步：创建目录**
```
skills/my-new-skill/
└── index.ts
```

**第 2 步：编写技能定义**

```typescript
import { SkillDefinition, SkillResult, SkillExecutionStep } from '../types';
import { callTool } from '../registry';

export const myNewSkill: SkillDefinition = {
  // 元数据（YAML frontmatter）
  metadata: {
    name: 'my-new-skill',
    description: '技能描述。Triggers: "关键词 1", "关键词 2"。Output: 输出什么',
    version: '1.0.0',
    author: '启源 AI',
    tags: ['tag1', 'tag2'],
    userInvocable: true,
    allowedTools: ['tool-id-1', 'tool-id-2'],
    metadata: {
      emoji: '🔧',
      pattern: 'tool-wrapper'
    }
  },
  
  // 参数定义
  params: [
    {
      name: 'param1',
      type: 'string',
      required: true,
      description: '参数描述',
      example: '示例值'
    }
  ],
  
  // 使用场景
  whenToUse: [
    '场景 1',
    '场景 2'
  ],
  
  // 示例
  examples: [
    {
      id: 'example-1',
      name: '示例 1',
      input: '用户输入',
      expectedOutput: '预期输出'
    }
  ],
  
  // 错误处理
  errorHandling: [
    {
      errorType: 'ERROR_TYPE',
      cause: '错误原因',
      solution: '解决方案'
    }
  ],
  
  // 执行函数
  async execute(
    params: Record<string, any>,
    steps?: SkillExecutionStep[]
  ): Promise<SkillResult> {
    // 调用工具
    const result = await callTool(
      'tool-id',
      { param1: params.param1 },
      steps || [],
      '步骤名称',
      '步骤描述'
    );
    
    if (result.success) {
      return {
        success: true,
        data: result.data,
        steps
      };
    } else {
      return {
        success: false,
        error: result.error,
        steps
      };
    }
  }
};

export default myNewSkill;
```

## 六、最佳实践

### 6.1 技能设计原则

1. **单一职责**：一个技能只做一件事
2. **清晰的 description**：决定技能何时被触发
3. **完整的示例**：至少 2 个示例 + 1 个边界案例
4. **错误处理**：覆盖常见失败场景
5. **参数验证**：必填参数要在 execute 中检查

### 6.2 工具使用规范

```typescript
// ✅ 好的做法：使用 callTool 自动记录步骤
const result = await callTool(
  'tool-id',
  params,
  steps || [],
  '步骤名称',
  '步骤描述'
);

// ❌ 不好的做法：直接调用工具，不记录步骤
const tool = getToolById('tool-id');
const result = await tool.execute(params);
```

### 6.3 错误处理

```typescript
try {
  // 参数验证
  if (!params.query) {
    return { success: false, error: '请提供搜索关键词', steps };
  }
  
  // 调用工具
  const result = await callTool(...);
  
  if (result.success) {
    return { success: true, data: result.data, steps };
  } else {
    return { success: false, error: result.error, steps };
  }
  
} catch (error: any) {
  return {
    success: false,
    error: error.message || '执行失败',
    steps
  };
}
```

## 九、常见问题

### Q1: 技能什么时候触发？
A: 由意图识别层决定，当用户输入匹配到技能的 `description` 中的关键词时触发。

### Q2: 如何调试技能？
A: 在浏览器控制台查看日志，技能执行时会输出详细日志。

### Q3: 技能执行失败怎么办？
A: 查看 `error` 字段和日志，检查参数是否正确、工具是否可用。

### Q4: 如何禁用某个技能？
A: 使用 `skillRegistry.setEnabled('skill-id', false)`。
