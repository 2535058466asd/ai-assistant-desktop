## 目录规则

- `design/`：UI、交互、视觉规范。
- `dev/`：当前活跃的功能实现计划、技术方案、阶段性开发记录。
- `completed/`：已经完成的开发计划和实现记录，只作回溯，不作为当前待办。
- 项目总览、安装和演示说明放根目录 `README.md`。
- 临时工具输出、日志、构建产物不要放入 docs。
