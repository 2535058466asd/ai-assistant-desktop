## 五、语音动画统一（P1）

### 问题
语音转文字（输入框麦克风）和语音对话模式（Header 语音按钮）的动画风格不统一。

### 统一方案
共享同一套颜色系统和动画风格：

| 状态 | 颜色 | 动画 |
|------|------|------|
| 录音中 | 红色 (#ef4444) | 脉冲波纹扩散 |
| 监听中 | 绿色 (#22c55e) | 呼吸闪烁 |
| 思考中 | 蓝色 (#3b82f6) | 脉冲缩放 |
| 播放中 | 橙色 (#f97316) | 弹跳动画 |

### 语音转文字（轻量）
- 麦克风图标变红色
- 图标周围加脉冲波纹
- 输入框边框变红色

### 语音对话模式（完整）
- 底部状态栏显示当前状态
- 状态指示点用对应颜色 + 动画
- 整体风格与语音转文字一致

### 涉及文件
- `InputArea.tsx` + `InputArea.module.css` — 语音转文字动画
- `Header.tsx` + `Header.module.css` — 语音对话按钮动画

## 六、搜索对话优化（P2）

### 参考
千问的搜索交互：点击搜索图标展开搜索面板，输入时实时过滤。

### 方案
- 搜索框已有，确认输入时**实时过滤**对话列表
- 搜索图标点击时自动聚焦搜索框
- 无匹配结果时显示"未找到相关对话"

### 涉及文件
- `Sidebar.tsx` — 搜索过滤逻辑确认/优化

## 七、侧边栏收起动画（P2）

### 现状
已有 `transform + width` 过渡动画，时长 0.3s。

### 优化
- 确保收起/展开动画平滑，无闪烁
- 收起时内容加 `opacity` 渐变（先淡出再收宽度）
- 与千问/智谱对比，确保不比它们差

### 涉及文件
- `Sidebar.module.css` — 动画微调
