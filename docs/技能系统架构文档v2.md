# 启源 AI - 技能系统架构文档 v2.0

## 📚 一、核心理念

### Tool（工具）= 基础能力单元
- 只做一件事
- 可被多个技能复用
- 类似"乐高积木"

### Skill（技能）= 完整功能
- 组合一个或多个工具
- 解决用户的实际问题
- 类似"乐高模型"

## 🔧 五、实现示例

### 示例 1：open-folder 技能（不使用工具）

```typescript
// skills/system-control/open-folder.ts

export const openFolderSkill: SkillDefinition = {
  metadata: {
    name: 'open-folder',
    description: '打开本地文件夹',
    category: 'system-control',
    userFacing: true,  // 👤 用户可见
  },
  
  async execute(params, steps) {
    const folderName = params.folderName;
    
    // 直接调用 Electron API（不需要工具）
    if (window.electronAPI?.systemOpenFolder) {
      const result = await window.electronAPI.systemOpenFolder(folderName);
      
      return {
        success: result.success,
        data: { message: result.message },
        steps: [{
          stepNumber: 1,
          name: '打开文件夹',
          success: true,
          output: `已打开 ${folderName} 文件夹`
        }]
      };
    }
  }
};
```

### 示例 2：search-web 技能（使用工具）

```typescript
// skills/search/search-web.ts

export const searchWebSkill: SkillDefinition = {
  metadata: {
    name: 'search-web',
    description: '后台网页搜索',
    category: 'search',
    userFacing: false,  // 🤖 AI 内部使用
  },
  
  allowedTools: ['network.searxng-search'],  // 使用工具
  
  async execute(params, steps) {
    // 调用工具获取数据
    const result = await callTool('network.searxng-search', {
      query: params.query
    }, steps);
    
    if (result.success) {
      // AI 处理数据后返回给用户
      return {
        success: true,
        data: {
          message: `找到 ${result.data.resultsCount} 条结果`,
          results: result.data.results  // 结构化数据
        },
        steps
      };
    }
  }
};
```

## 💡 七、关键决策点

### Q1：用户技能和 AI 技能是否分开？

**建议：代码放一起，用标记区分**

```typescript
metadata: {
  userFacing: true,  // true = 用户技能, false = AI 技能
}
```

**原因**：
- ✅ 统一管理，方便维护
- ✅ 可以动态切换（某些场景下 AI 也可以调用用户技能）
- ❌ 分开会导致代码重复和维护困难

### Q2：什么时候需要创建工具？

**规则**：
- ✅ **需要创建工具**：功能会被多个技能复用，或者逻辑较复杂
- ❌ **不需要工具**：只是一次性调用 API，逻辑简单

**示例**：
- ✅ `network.http-get` → 多个技能都需要发送 HTTP 请求
- ❌ `system.open-folder` → 只有 open-folder 技能用，且只是调用一个 API

### Q3：如何处理 SearXNG 问题？

**短期方案**：改用 DuckDuckGo API（免费、无需配置）

**长期方案**：
- 配置 SearXNG 允许 JSON API
- 或者自建搜索服务
